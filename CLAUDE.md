# 프로젝트 규칙 (QA TOOL)

> **글로벌 규칙은 `~/.claude/CLAUDE.md` 참조**
> 이 파일에는 이 프로젝트에만 적용되는 규칙만 기록합니다.

---

## 0. 프로젝트 핵심 관점

> **이 도구는 다른 프로젝트 위에 얹혀서 동작하는 QA 검수 도구다. 자체 서비스가 아니다.**

- **이 서비스는 무엇인가?** — 브라우저에서 UI 요소를 클릭/선택하여 시각적 QA 피드백을 남기고, 수정 요청 및 재검수까지 처리하는 로컬 QA 검수 도구
- **사용자는 누구인가?** — 비개발자 대표(연진) 1인, 클로드 코드와 협업하여 웹 프로젝트를 개발하는 환경
- **왜 만드는가?** — QA 파이프라인의 수작업 단계를 줄여 효율적인 검수/재검수 사이클 확보
- **이 서비스가 아닌 것은?** — 독립 실행 웹앱이 아님. 다른 프로젝트의 HTML에 script 태그로 삽입하는 도구

---

## 1. 기술 스택

- 프론트엔드: 순수 JavaScript (단일 파일, IIFE 패턴)
- 백엔드: 없음
- DB: 없음 (localStorage만 사용)
- 배포: 없음 (로컬 개발 도구, script 태그 삽입)

---

## 2. 파일 구조 (프로젝트 고유)

```
qa-feedback.js     — 핵심 코드 (단일 파일)
docs/PRD.md        — PRD v2.0
docs/DEV-REQUEST.md — 개발 요청서
docs/QA.md         — QA 이슈 트래커
hooks/             — 검증 스크립트
```

---

## 3. 프로젝트 고유 규칙

- 외부 의존성 금지 — npm, CDN, 프레임워크 사용 안 함
- CSS 네임스페이스 필수 — `.qa-feedback-`, `.qa-settings-`, `.qa-session-` 접두사
- z-index 범위: 99980~100000
- 전역 오염 금지 — IIFE 패턴 유지
- 기존 페이지 스타일과 충돌 방지 — 모든 스타일을 네임스페이스 안에서 정의
- 저장 구조 변경 시 하위 호환 고려 — 기존 localStorage 데이터가 깨지지 않도록
