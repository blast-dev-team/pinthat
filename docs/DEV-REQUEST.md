### DEV-001: 영속 저장 (sessionStorage → localStorage)
- **상태**: 🔵 개발완료
- **요청일**: 2026-03-07
- **서비스 본질 연결**: 탭을 닫아도 QA 피드백이 유지되어야 다른 업무를 하다 돌아올 수 있음
- **상세 스펙**:
  - `qa-feedback.js` 파일 수정
  - `STORAGE_KEY` 상수 (22행): 기존 유지 (`'qa-feedbacks-' + location.pathname`)
  - `saveFeedbacks()` 함수 (24~35행): `sessionStorage` → `localStorage`로 변경
  - `restoreFeedbacks()` 함수 (37~61행): `sessionStorage` → `localStorage`로 변경
  - `resetAll()` 함수 (804~820행): `sessionStorage.removeItem` → `localStorage.removeItem`으로 변경
  - 단축키 저장(`qa-shortcuts`)은 이미 localStorage라 변경 없음
  - **주의**: 저장/복원 데이터 구조는 기존과 동일하게 유지 (`{ feedbacks: [...], nextId: N }`)
- **의존 관계**: 없음 (첫 번째 작업)
- **테스트 방법**:
  1. `demo.html` 열기 → Alt+Q → 요소 3개에 피드백 남기기
  2. 탭 완전히 닫기
  3. `demo.html` 다시 열기 → 피드백 3개 자동 복원 확인
  4. 초기화(R) → localStorage에서 데이터 삭제 확인 (DevTools > Application > Local Storage)
  5. 기존 기능(요소/텍스트/영역 선택, 마크다운 출력, 단축키 설정 등) 모두 정상 동작 확인
- **구현 메모**: `saveFeedbacks()`, `restoreFeedbacks()`, `resetAll()` 3곳에서 `sessionStorage` → `localStorage`로 변경. 저장 키/데이터 구조는 기존 유지.
- **검수 결과**: (코워커 작성)

### DEV-002: 멀티페이지 세션 관리
- **상태**: 🔵 개발완료
- **요청일**: 2026-03-07
- **서비스 본질 연결**: 여러 페이지의 피드백을 세트로 관리하여 재검수 시 한번에 불러올 수 있어야 함
- **상세 스펙**:
  - `qa-feedback.js` 파일에 세션 관리 기능 추가

  **1) 데이터 구조** (localStorage 키: `qa-sessions`):
  ```json
  {
    "sessions": [
      {
        "id": "session-1709812345678",
        "name": "인플루언서 프로필 QA",
        "page": "/demo.html",
        "url": "http://localhost/demo.html",
        "createdAt": "2026-03-07T15:30:00",
        "status": "open",
        "feedbacks": [피드백 배열 — 기존 STATE.feedbacks 구조 그대로],
        "nextId": 4
      }
    ]
  }
  ```

  **2) QA 패널에 버튼 2개 추가** (기존 "단축키 설정" 버튼 위에):
  ```
  ─────────────────  (구분선)
  💾 세션 저장
  📂 세션 불러오기
  ─────────────────  (구분선)
  ⚙️ 단축키 설정     (기존)
  ```

  **3) "세션 저장" 클릭 시**:
  - 피드백이 0건이면 alert("저장할 피드백이 없습니다.")
  - 피드백 1건 이상이면 → 이름 입력 팝업 표시
  - 기본 이름: 페이지 파일명 + 날짜 (예: "demo.html — 2026-03-07")
  - "저장" 클릭 → localStorage `qa-sessions`에 추가
  - 저장 완료 토스트 메시지: "세션이 저장되었습니다."

  **4) "세션 불러오기" 클릭 시**:
  - localStorage에서 세션 목록 조회
  - 세션 없으면 alert("저장된 세션이 없습니다.")
  - 세션 있으면 → 목록 팝업 표시:
    ```
    ┌─────────────────────────────────┐
    │ 📂 저장된 세션                    │
    │                                  │
    │ ▸ 인플루언서 프로필 QA            │
    │   3건 · 2026-03-07 · open       │
    │   [불러오기] [재검수] [삭제]       │
    │                                  │
    │ ▸ 메인페이지 QA                   │
    │   5건 · 2026-03-06 · closed     │
    │   [불러오기] [재검수] [삭제]       │
    │                                  │
    │                        [닫기]     │
    └─────────────────────────────────┘
    ```
  - "불러오기" → 현재 피드백 초기화 후 세션 피드백으로 교체 + 핀 복원
  - "삭제" → confirm 후 세션 삭제
  - "재검수" → DEV-003에서 구현 (이 단계에서는 버튼만 비활성 표시)

  **5) 세션 자동 정리**:
  - 30일 이상 된 세션은 불러오기 시 자동 삭제 안내

- **의존 관계**: DEV-001 이후
- **테스트 방법**:
  1. 피드백 3개 남기기 → "세션 저장" → 이름 입력 → 저장 확인
  2. 초기화 → 피드백 모두 사라짐
  3. "세션 불러오기" → 목록에 저장한 세션 표시 확인
  4. "불러오기" 클릭 → 핀 3개 복원 확인
  5. 탭 닫기 → 다시 열기 → "세션 불러오기" → 여전히 존재 확인
  6. "삭제" → 세션 삭제 확인
  7. 세션 저장을 2번 이상 하여 목록에 여러 세션 표시 확인
- **구현 메모**: 세션 CRUD 함수 (`getSessionsData`, `saveSessionsData`, `cleanOldSessions`) 추가. QA 패널에 "세션 저장"/"세션 불러오기" 버튼 추가. 세션 저장 시 이름 입력 팝업, 세션 목록 팝업 구현. 30일 이상 세션 자동 정리. 토스트 메시지 표시.
- **검수 결과**: (코워커 작성)

### DEV-003: 재검수 모드
- **상태**: 🔵 개발완료
- **요청일**: 2026-03-07
- **서비스 본질 연결**: 수정 완료 후 이전 피드백 위치를 자동 복원하여 수정 여부를 빠르게 확인해야 함. 이 기능이 없으면 재검수 시 다른 업무를 할 수 없음
- **상세 스펙**:
  - `qa-feedback.js` 파일에 재검수 모드 추가

  **1) 재검수 진입**:
  - 세션 목록에서 "재검수" 버튼 클릭
  - 세션 status를 "reviewing"으로 변경
  - 기존 피드백(있으면) 임시 저장 후, 재검수 세션으로 전환

  **2) 재검수 핀 UI**:
  - 기존 빨간 핀 대신 **재검수 전용 핀** 표시:
    ```
    [노란 테두리 + 번호 뱃지]
    뱃지 클릭 시 → 재검수 팝업:
    ┌──────────────────────────┐
    │ ① [UI] 제목 폰트 크기     │
    │ 원본 피드백: "크기 줄여야 함" │
    │ ─────────────────────── │
    │ ◉ ✅ 수정됨               │
    │ ○ ❌ 미수정               │
    │ ─────────────────────── │
    │ 메모: [________________]  │
    │           [확인]          │
    └──────────────────────────┘
    ```

  **3) 핀 복원 로직**:
  - 저장된 `selector`로 `document.querySelector()` 실행
  - 찾은 경우 → 해당 위치에 재검수 핀 (노란 테두리)
  - 못 찾은 경우 → 뱃지를 회색으로 표시 + "⚠️ 요소를 찾을 수 없음" 텍스트
  - 영역 선택(`areaBox`)인 경우 → 저장된 좌표로 영역 핀 복원

  **4) 재검수 상태 관리**:
  - 각 피드백에 `reviewStatus` 필드 추가: `null` | `"fixed"` | `"not-fixed"`
  - 각 피드백에 `reviewNote` 필드 추가: 재검수 시 추가 메모
  - 상태 변경할 때마다 세션 자동 저장

  **5) 재검수 완료**:
  - QA 패널에 "재검수 완료" 버튼 표시 (재검수 모드일 때만)
  - 클릭 시 → 마크다운 출력 (기존 출력 형식 확장):
    ```markdown
    # QA 재검수 리포트 — demo.html
    > 원본 검수일: 2026-03-07 15:30
    > 재검수일: 2026-03-07 18:00
    > 결과: 2건 수정됨 / 1건 미수정

    ---

    ## ① [UI] H1 — hero
    - **요소**: `section.hero h1`
    - **원본 피드백**: 제목 폰트 크기 줄여야 함
    - **재검수 결과**: ✅ 수정됨
    - **재검수 메모**: 32px → 24px로 변경 확인

    ## ② [기능] BUTTON — modal
    - **요소**: `.btn-primary`
    - **원본 피드백**: 클릭 시 에러 발생
    - **재검수 결과**: ❌ 미수정
    - **재검수 메모**: 여전히 alert 에러 발생
    ```
  - 세션 status를 "closed"로 변경
  - 클립보드에 복사

  **6) 재검수 모드 종료**:
  - "재검수 완료" 또는 Escape로 종료
  - 재검수 핀 제거, 일반 모드로 복귀

- **의존 관계**: DEV-002 이후
- **테스트 방법**:
  1. 피드백 3개 남기기 → 세션 저장
  2. 초기화
  3. (수정 시뮬레이션: demo.html에서 텍스트 직접 변경)
  4. "세션 불러오기" → "재검수" 클릭
  5. 노란 핀 3개 표시 확인
  6. 각 핀 클릭 → 재검수 팝업 표시 확인
  7. ✅/❌ 체크 + 메모 입력
  8. "재검수 완료" → 마크다운 출력에 재검수 결과 포함 확인
  9. 클립보드 복사 후 내용 확인
  10. 세션 status가 "closed"로 변경되었는지 확인
- **구현 메모**: STATE에 `reviewMode`, `reviewSessionId`, `savedFeedbacksBeforeReview` 추가. 재검수 전용 핀 UI (노란 테두리, 상태별 뱃지 색상: 노란=미확인, 초록=수정됨, 빨강=미수정, 회색=요소 못 찾음). 재검수 팝업에 라디오 버튼(수정됨/미수정) + 메모 입력. 상태 변경 시 세션 자동 저장. 재검수 완료 시 마크다운 리포트 생성 + 클립보드 복사 + 세션 closed. Escape로 재검수 모드 종료 가능.
- **검수 결과**: (코워커 작성)

### DEV-004: 재검수 완료 후 세션 삭제 확인 팝업
- **상태**: 🔵 개발완료
- **요청일**: 2026-03-07
- **서비스 본질 연결**: 재검수 완료 = 검수 사이클 종료이므로, 불필요한 세션 정리를 자연스럽게 유도해야 함
- **상세 스펙**:
  - "재검수 완료" 클릭 시 기존 동작 유지 (마크다운 리포트 생성 + 클립보드 복사)
  - 리포트 복사 완료 후 → **confirm 팝업 표시**: "세션을 삭제하시겠습니까?"
  - "삭제" → 해당 세션을 `qa-sessions`에서 제거
  - "유지" → 세션 status를 "closed"로 유지 (기존 동작)
- **의존 관계**: DEV-003 이후
- **테스트 방법**:
  1. 재검수 모드 진입 → 모든 핀 체크 → "재검수 완료" 클릭

  3. "세션을 삭제하시겠습니까?" 팝업 표시 확인
  4. "삭제" 선택 → 세션 불러오기에서 해당 세션 사라진 것 확인
  5. (다시 테스트) "유지" 선택 → 세션이 closed 상태로 남아있는 것 확인

### DEV-005: 재검수 취소 버튼 추가
- **상태**: 🔵 개발완료
- **요청일**: 2026-03-07
- **서비스 본질 연결**: 재검수 도중 중단하고 싶을 때 나갈 방법이 없으면 사용자가 답답함. Escape 키는 존재하지만 UI에 표시 안 됨
- **상세 스펙**:
  - QA 패널에서 "재검수 완료" 버튼 위에 **"재검수 취소"** 버튼 추가
  - 스타일: 회색 계열 (완료 버튼의 주황색과 구분)
  - 클릭 시 동작:
    - 재검수 핀 전부 제거
    - 재검수 진입 전 기존 피드백 복원 (STATE.savedFeedbacksBeforeReview)
    - 세션 status를 "open"으로 되돌림 (reviewing → open)
    - 재검수 모드 종료
    - 토스트: "재검수가 취소되었습니다."
  - Escape 키에도 동일 동작 연결 (기존 exitReviewMode와 동일)
- **의존 관계**: DEV-003 이후
- **테스트 방법**:
  1. 세션 저장 → 재검수 진입 → 핀 1~2개만 체크
  2. "재검수 취소" 클릭 → 재검수 핀 사라지고 원래 상태 복귀 확인
  3. 세션 불러오기 → 해당 세션이 "open" 상태인지 확인
  4. Escape 키로도 동일하게 취소되는지 확인

### DEV-006: 마크다운 리포트 가져오기 (역방향 복원)
- **상태**: 🔵 개발완료
- **요청일**: 2026-03-07
- **서비스 본질 연결**: QA 리포트를 클로드 코드에 보낸 뒤, 수정 후 같은 리포트를 붙여넣어 재검수 핀을 복원할 수 있어야 QA→수정→재검수 루프가 완성됨
- **상세 스펙**:
  - QA 패널에 **"📋 마크다운 가져오기"** 버튼 추가 (세션 불러오기 아래)
  - 클릭 시 → 텍스트 입력 팝업 표시 (textarea, 넉넉한 크기)
  - 사용자가 마크다운 리포트를 붙여넣고 "가져오기" 클릭
  - 파싱 대상 필드:
    - `## ① [UI] DIV — body` → 번호, 피드백타입, 태그명, 섹션
    - `- **요소**: \`selector\`` → CSS selector (핀 복원에 사용)
    - `- **피드백**: 내용` → 피드백 텍스트
    - `- **영역 좌표**: {x, y, w, h}` → 영역 선택 복원 (있을 경우)
  - 파싱 후:
    - 기존 피드백 초기화
    - 각 항목의 selector로 `document.querySelector()` 실행
    - 찾은 요소에 핀 복원
    - 못 찾은 요소는 "⚠️ 요소를 찾을 수 없음" 표시
    - 토스트: "N건의 피드백을 가져왔습니다. (M건 복원 성공)"
  - 가져온 후 바로 **재검수 모드로 자동 진입** (선택 옵션)
- **의존 관계**: DEV-003 이후
- **테스트 방법**:
  1. 핀 3개 찍기 → 마크다운 출력 (E) → 텍스트 복사
  2. 초기화 → "마크다운 가져오기" 클릭 → 복사한 텍스트 붙여넣기
