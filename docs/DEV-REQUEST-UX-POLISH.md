## 수정 1: GitHub Issue 전송 방식 개선

### 1-1. 전송 모달에 제목 입력란 추가

마크다운 출력 모달에서 "GitHub Issue 전송" 클릭 시, 바로 전송하지 말고 **전송 확인 모달**을 띄운다.

```
┌──────────────────────────────────────────┐
│  🚀 GitHub Issue 전송                [X] │
├──────────────────────────────────────────┤
│                                          │
│  이슈 제목 (선택)                        │
│  ┌──────────────────────────────────┐    │
│  │ placeholder: 미입력 시 자동 생성  │    │
│  └──────────────────────────────────┘    │
│                                          │
│  전송 범위                               │
│  ○ 전체 피드백 (3건)                     │
│  ○ 선택한 피드백만                       │
│                                          │
│  (선택한 피드백만 고른 경우)             │
│  ☑ ① [텍스트] "3" — body               │
│  ☑ ② [UI] 버튼 색상 — header            │
│  ☐ ③ [위치이동] 사이드바 — main          │
│                                          │
│         [취소]  [전송]                   │
└──────────────────────────────────────────┘
```

### 1-2. 제목 로직

- **제목 미입력 시**: 기존 자동 생성 형식 유지
  - 전체: `[QA] index — 피드백 3건 (2026-03-31)`
  - 개별(1건): `[QA] index — 피드백 1건 (2026-03-31)`
- **제목 입력 시**: 입력값 그대로 이슈 제목으로 사용
  - 예: 사용자가 "로그인 페이지 전체 수정 필요" 입력 → 이슈 제목: `로그인 페이지 전체 수정 필요`

### 1-3. 전송 범위

- **전체 피드백**: 현재 페이지의 모든 피드백을 하나의 이슈로 전송 (기존 동작)
- **선택한 피드백만**: 체크박스로 원하는 피드백만 골라서 전송
  - 체크박스 목록에 피드백 번호 + 유형 + 요약 표시
  - 기본: 전체 체크 상태
  - 선택된 피드백만 이슈 본문에 포함

### 1-4. 개별 전송 (1건씩)

- 전송 범위에서 피드백 1건만 체크하면 자연스럽게 개별 전송이 됨
- 별도 UI 불필요

## 수정 3: 이슈 현황 페이지네이션

### 3-1. 5개씩 페이지네이션

- 이슈 목록을 5개씩 표시
- 하단에 [◀ 이전] [▶ 다음] 버튼 추가
- 현재 페이지 표시: "1 / 3 페이지"
- API 호출: `per_page=5&page={n}`

### 3-2. 이슈 현황 모달 디자인 업데이트

```
┌──────────────────────────────────────────────┐
│  📋 GitHub 이슈 현황 (12건)           [X]    │
│  [전체] [Open] [Closed]                      │
├──────────────────────────────────────────────┤
│ 🟢 이슈1...                 [GitHub ↗]       │
│ 🟢 이슈2...                 [GitHub ↗]       │
│ ✅ 이슈3...          [재검수] [GitHub ↗]     │
│ ✅ 이슈4...          [재검수] [GitHub ↗]     │
│ 🟢 이슈5...                 [GitHub ↗]       │
├──────────────────────────────────────────────┤
│       [◀ 이전]  1 / 3  [▶ 다음]             │
└──────────────────────────────────────────────┘
```

### 3-3. 완료된 이슈 아카이브 (숨기기)

- Closed 이슈 항목에 [재검수] 옆에 [아카이브] 버튼 추가
- 아카이브 클릭 시: chrome.storage.local에 해당 이슈 번호를 아카이브 목록에 저장
- 아카이브된 이슈는 기본적으로 목록에서 숨김
- 필터 탭에 [아카이브] 탭 추가 → 숨긴 이슈 확인 가능
- 아카이브에서 [복원] 버튼으로 다시 목록에 표시 가능

## 수정 4: 출력 모달 단순화

### 4-1. 상세도 레벨 제거

- Compact / Standard / Detailed / Forensic 선택 버튼 제거
- **Detailed 레벨로 고정** (요소 + 텍스트 + 피드백 + 위치 좌표 + 추가 요소 포함)
- STATE.detailLevel 변수 제거 → generateMarkdown() 호출 시 항상 'detailed' 사용
- Forensic 레벨의 CSS 스타일 출력은 제거 (과도한 정보)

### 4-2. DEV-REQUEST 탭 제거

- 출력 모달에서 "DEV-REQUEST" 탭 제거
- 재검수 모달의 "미수정 DEV-REQUEST" 탭도 제거
- DEV-REQUEST 생성 함수(generateDevRequest 등) 제거
- 출력 모달은 **마크다운 탭 하나만** 유지
- 탭 UI 자체를 제거하고 바로 마크다운 내용 표시

### 4-3. 출력 모달 최종 구조

```
┌──────────────────────────────────────────┐
│  출력 (3건)                         [X]  │
├──────────────────────────────────────────┤
│                                          │
│  (마크다운 내용 바로 표시)               │
│                                          │
├──────────────────────────────────────────┤
│  [닫기]  [클립보드 복사]  [GitHub Issue]  │
└──────────────────────────────────────────┘
```

### 테스트 체크리스트
- [ ] 상세도 선택 버튼 (Compact/Standard/Detailed/Forensic) 없음
- [ ] DEV-REQUEST 탭 없음
- [ ] 마크다운 내용이 바로 표시됨
- [ ] 위치 좌표(bbox) 정보가 포함됨
- [ ] CSS 스타일(forensic) 정보는 미포함
- [ ] 클립보드 복사 + GitHub Issue 버튼 정상 동작

## 주의사항

- CSS 네임스페이스 규칙 준수
- z-index 범위: 99980~100000
- 전송 확인 모달은 마크다운 출력 모달 위에 overlay로 표시
- 아카이브 데이터는 chrome.storage.local에 저장 (키: `qa-archived-issues`)
- 기존 전송 로직(ensureLabelExists, recordIssueHistory 등)은 유지
- 토큰은 절대 console.log 하지 않음
