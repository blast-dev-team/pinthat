# DEV-REQUEST: 자유 위치 이동 — 좌표 기반으로 변경

> 작성일: 2026-03-31
> 우선순위: 중간
> 대상 파일: `qa-tool-extension/content-script.js`

---

## 배경

현재 "자유 위치 이동"은 소스 요소 → 목적지 요소를 모두 컴포넌트 단위로 선택하는 구조.
하지만 "자유 이동"이라는 이름답게, 목적지는 컴포넌트에 스냅되지 않고 **화면의 아무 위치**를 지정할 수 있어야 함.

정리:
- **컴포넌트 이동**: 소스 요소 선택 → 방향(위/아래/좌/우) = 기존 그대로 유지
- **자유 위치 이동**: 소스 요소 선택 → 목적지는 **화면 아무 곳 클릭** (컴포넌트에 스냅하지 않음)

---

## 수정 사항

### 1. 자유 이동 목적지 선택 방식 변경

- 소스 요소 선택 후 → 목적지 선택 모드 진입
- **기존**: hover 시 요소 하이라이트 → 클릭으로 목적지 요소 선택
- **변경**: hover 하이라이트 OFF → 화면 아무 곳 클릭 → 클릭한 좌표를 기록
- 커서: `crosshair`로 변경하여 "위치 지정 모드"임을 시각적으로 표시

### 2. 클릭 좌표 처리

- 클릭 시 `event.clientX`, `event.clientY` 기록
- 클릭 좌표에서 가장 가까운 요소(element)를 `document.elementFromPoint(x, y)`로 참조
- 피드백 데이터에 저장:
  ```javascript
  {
    moveType: 'free',
    moveTarget: {
      x: event.pageX,        // 스크롤 포함 절대 좌표
      y: event.pageY,
      nearestSelector: '가장 가까운 요소의 CSS 셀렉터',
      description: '(가장 가까운 요소) 근처'
    }
  }
  ```

### 3. 목적지 표시 (시각적)

- 클릭한 위치에 작은 마커(●) 표시 (빨간색, 12px 원)
- 소스 요소에서 마커까지 점선 연결 (optional, 가능하면)
- 오버레이에는 소스 요소만 보라색으로 표시 (기존과 동일)

### 4. 마크다운 출력 변경

- 기존: `이동 목적지: div > main > ...` (요소 셀렉터)
- 변경: `이동 목적지: (CSS셀렉터) 근처 (x, y 좌표)` 형태
- 예시: `이동 목적지: .sidebar 근처로 이동 요청`

### 5. 메모 입력 유도

- 자유 이동은 좌표만으로는 의도가 불분명할 수 있으므로
- 메모 입력 시 placeholder를 "예: 사이드바 아래로 옮겨주세요"로 변경
- 메모가 비어있으면 `showToast('자유 이동은 메모를 입력해주세요.')`로 안내 (저장은 허용)

---

## 변경하지 않는 것

- 컴포넌트 이동 (방향 기반) → 기존 그대로
- 소스 요소 선택 방식 → 기존 그대로 (컴포넌트 스냅 유지)
- 오버레이 색상/스타일 → 기존 그대로

---

## 테스트 체크리스트

- [ ] 위치이동 → 자유 위치 이동 선택 → 소스 요소 클릭
- [ ] 목적지 모드에서 hover 시 요소 하이라이트가 안 뜨는지 확인
- [ ] 커서가 crosshair로 변경되는지 확인
- [ ] 빈 공간 클릭 → 마커 표시 + 피드백 저장됨
- [ ] 컴포넌트 위 클릭 → 해당 위치 좌표로 저장 (컴포넌트 선택이 아님)
- [ ] 마크다운 출력에서 "근처" 형태로 표시되는지 확인
- [ ] GitHub Issue 전송 시 자유 이동 내용이 정상 포함되는지 확인

---

## 주의사항

- CSS 네임스페이스 규칙 준수
- 기존 STATE.moveMode, STATE.moveSource 구조 활용
- 좌표는 pageX/pageY (스크롤 포함) 사용 — clientX/clientY는 뷰포트 기준이라 스크롤 시 어긋남
- z-index 범위: 99980~100000
