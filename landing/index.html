Features

QA를 더 빠르고, 더 정확하게

스크린샷 찍고 문서 작성하는 시간은 끝. 브라우저 위에서 바로 핀을 꽂으세요.

📌

클릭 한 번으로 피드백

UI 요소를 클릭하면 바로 피드백 입력창이 뜹니다. 셀렉터, 위치, 텍스트를 자동 캡처하니 "어디 말하는 거예요?"는 이제 없어요.

🔀

위치 이동 피드백

"이 버튼 왼쪽으로 옮겨줘" — 방향 지정부터, 화면 아무 곳 클릭으로 목적지를 찍는 자유 이동까지. 개발자가 바로 이해하는 포맷.

🐙

GitHub Issue 자동 전송

피드백을 GitHub Issue로 바로 보내세요. 제목 지정, 선택/전체 전송, 라벨 자동 부여. 복사-붙여넣기는 이제 안녕.

🔄

재검수 워크플로우

이슈 현황에서 Closed된 이슈를 재검수. 안 고쳐졌으면 한 클릭으로 Reopen + 코멘트까지. 완벽한 QA 사이클.

How it works

3단계로 끝나는 QA

설치하고, 꽂고, 보내세요. 그게 전부입니다.

1

핀 꽂기

QA 모드를 켜고 문제 있는 요소를 클릭하세요. UI, 기능, 텍스트, 위치 이동 중 유형을 선택합니다.

→
2

피드백 작성

무엇을 바꿔야 하는지 적으세요. 요소의 위치, 텍스트, 셀렉터는 자동으로 기록됩니다.

→
3

GitHub 전송

피드백을 GitHub Issue로 한 번에 전송하세요. 개발자는 Issue에서 바로 작업을 시작할 수 있습니다.
