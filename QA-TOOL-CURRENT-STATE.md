### 1-2. 키보드 단축키 (기본값)

| 단축키 | 동작 | 조건 |
|--------|------|------|
| `⌥Q` (Option/Alt + Q) | 검수 모드 ON/OFF | 항상 |
| `Escape` | 피드백 팝업 / 출력 모달 닫기 | 항상 |
| `1` | 요소 선택 모드 | 검수 ON일 때만 |
| `2` | 텍스트 선택 모드 | 검수 ON일 때만 |
| `3` | 영역 선택 모드 | 검수 ON일 때만 |
| `E` | 마크다운 출력 | 피드백 1건 이상 (모드 무관) |
| `R` | 전체 초기화 | 검수 ON일 때만 |
| `Shift + 클릭` | 다중 선택 | 검수 ON일 때만 |
| `Ctrl/⌘ + Enter` | 피드백 저장 | 피드백 입력 중 |

**단축키 비활성화 조건**:
- `textarea`, `input`, `select`에 포커스 시 → 모든 단축키 무시
- 단축키 설정 팝업(`.qa-settings-overlay`) 열려있을 때 → 모든 단축키 무시

### 1-3. 단축키 설정 팝업

**진입**: QA 패널 하단 `⚙️ 단축키 설정` 버튼 클릭

**UI 구조**:
```
┌──────────────────────────────────────┐
│ ⚙️ 단축키 설정                        │
│                                       │
│ 검수 ON/OFF:  [⌥Q]  [변경]           │
│ 요소 선택:    [1]   [변경]            │
│ 텍스트 선택:  [2]   [변경]            │
│ 영역 선택:    [3]   [변경]            │
│ 마크다운 출력: [E]   [변경]            │
│ 초기화:       [R]   [변경]            │
│                                       │
│  [기본값 복원]   [닫기]      [저장]     │
└──────────────────────────────────────┘
```

**동작 규칙**:

| 요소 | 동작 |
|------|------|
| **[변경]** | 클릭 → "키 입력..." 대기 → 키 입력 → 표시만 (적용 안 됨) |
| **[저장]** | 드래프트 → `shortcutKeys`에 적용 + `localStorage`에 저장 + 패널 힌트 업데이트 + 팝업 닫기 |
| **[닫기]** | 변경 사항 버리고 팝업만 닫기 |
| **[기본값 복원]** | 드래프트를 DEFAULT_KEYS로 되돌림 (팝업 유지, [저장]으로 확정) |

**기술 구현**:
- 드래프트 패턴: `draft = JSON.parse(JSON.stringify(shortcutKeys))` 사본 사용
- 수정자 키 지원: `{ alt: true, key: 'q' }` 오브젝트 형식
- 수정자 전용 키 무시: `['Alt','Control','Shift','Meta']` 단독 입력 무시
- `Escape` 입력 시 → 키 변경 취소 (대기 상태 해제)
- 저장 시 `localStorage('qa-shortcuts')`에 저장 → 새로고침/재방문 시 유지
- backdrop 클릭 → 팝업 닫기 (setTimeout 100ms 지연으로 열림 직후 닫힘 방지)
- 배경 이벤트 차단: `mousedown/mouseup/mousemove`는 `stopPropagation`만, `click`만 닫기 처리

### 1-4. 피드백 팝업 드래그 이동

- 팝업 헤더(`.qa-feedback-popup-header`)에 `cursor: move` 적용
- `makePopupDraggable(popup)` 함수로 신규/편집 팝업 모두 드래그 가능
- 뷰포트 경계 체크: `Math.max(0, ...)` → 화면 밖으로 나가지 않음
- 선택 표시(하이라이트/마커) 위치는 변하지 않음 — 팝업만 이동
- 뷰포트 아래로 잘리는 경우: 초기 위치를 `window.innerHeight - 300`으로 보정

### 1-5. sessionStorage 자동 저장

- **저장 키**: `'qa-feedbacks-' + location.pathname` (페이지별 독립)
- **저장 시점**: 피드백 추가/수정/삭제/Shift+클릭 시마다 `saveFeedbacks()` 호출
- **저장 내용**: `{ feedbacks: [...], nextId: N }` (DOM 참조 `el`은 `null`로 제외)
- **복원 시점**: Init 단계의 마지막 (`restoreFeedbacks()`)
- **복원 방법**: selector로 DOM 요소 재검색 → `addOverlay()`로 빨간 테두리 + 번호 뱃지 복원
- **초기화 시**: `sessionStorage.removeItem(STORAGE_KEY)` 호출
- **자동 삭제**: 탭/창 닫으면 `sessionStorage` 특성상 자동 삭제 (QA 세션 단위에 적합)

## 2. 초기화 시퀀스 (Init)

```javascript
loadShortcuts();      // localStorage에서 사용자 단축키 복원
injectStyles();       // CSS 스타일 주입 (모든 QA UI 스타일)
buildPanel();         // QA 패널 DOM 생성 + 이벤트 바인딩
buildSettings();      // ⚙️ 버튼에 설정 팝업 onclick 등록
updateHints();        // 패널 버튼에 단축키 힌트 텍스트 반영
restoreFeedbacks();   // sessionStorage에서 피드백 복원 + 오버레이 재생성
```
