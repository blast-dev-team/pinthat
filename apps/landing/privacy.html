PinThat

Privacy Policy

Last updated: March 31, 2026

1. Overview

PinThat ("we", "our", "the Extension") is a Chrome Extension that provides visual QA feedback tools for web projects. We are committed to protecting your privacy. This policy explains what data we collect, how we use it, and your rights.

2. Data We Collect

We do not collect any personal data.

All data created by PinThat is stored locally on your device using Chrome's chrome.storage.local API. This includes:

  • QA feedback entries (element selectors, feedback text, positions)
  • Session data (saved QA sessions)
  • User preferences (keyboard shortcuts, settings)
  • GitHub settings (repository info, Personal Access Token)

None of this data is transmitted to our servers. We do not operate any servers that receive user data.

3. GitHub API Communication

When you use the GitHub integration feature (Pro), PinThat communicates directly with GitHub's API (https://api.github.com) using your Personal Access Token. This communication occurs:

  • To verify repository access (connection test)
  • To create GitHub Issues from your QA feedback
  • To query issue status (open/closed)
  • To reopen issues and add comments during re-review

Your GitHub token is stored exclusively in chrome.storage.local within the Extension's sandboxed environment. It is never exposed to web pages, logged to console, or sent to any server other than GitHub's API.

4. Data We Do NOT Collect

  • No personal information (name, email, phone)
  • No browsing history
  • No analytics or tracking data
  • No cookies
  • No advertising data
  • No usage statistics

5. Third-Party Services

PinThat does not use any third-party analytics, advertising, or tracking services. The only external service communication is with GitHub's API, initiated solely by your explicit action (clicking "Send" or "Connection Test").

6. Permissions

PinThat requires the following Chrome permissions:

  • activeTab — To interact with the current web page for QA feedback
  • storage — To save feedback, sessions, and settings locally
  • clipboardWrite — To copy markdown output to clipboard
  • host_permissions (api.github.com) — To communicate with GitHub API for issue creation

7. Data Retention & Deletion

Since all data is stored locally on your device, you have full control. You can delete all PinThat data by:

  • Using the "Reset" button in the Extension panel
  • Uninstalling the Extension (removes all associated data)
  • Clearing Extension data via Chrome settings

8. Children's Privacy

PinThat is not directed at children under the age of 13. We do not knowingly collect any data from children.

9. Changes to This Policy

We may update this privacy policy from time to time. Any changes will be reflected on this page with an updated "Last updated" date.
