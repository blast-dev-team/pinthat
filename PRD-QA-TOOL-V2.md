## 1. 배경

### 현재 상황
비개발자(대표)가 클로드 코드와 협업하여 웹 프로젝트를 개발하고 있음.
QA 검수 도구(`qa-feedback.js`)를 자체 개발하여 marketing-automation 프로젝트에 적용 중.

### 현재 QA 파이프라인
```
개발 완료
  → 브라우저에서 페이지 열기
  → QA 모드 ON (Alt+Q)
  → 문제 요소 클릭/텍스트 선택/영역 선택
  → 피드백 입력 (UI/기능/텍스트 유형 분류)
  → 마크다운 자동 생성 + 클립보드 복사
  → 클로드 코드에 붙여넣기
  → 수정
```

### 현재 도구 스펙 요약 (qa-feedback.js v1)
- 파일: `assets/js/qa-feedback.js` (1,017줄, 순수 JS, 외부 의존성 없음)
- IIFE 패턴, 전역 오염 없음
- CSS 네임스페이스: `.qa-feedback-`, `.qa-settings-`

#### 구현된 기능 (13건)
| # | 기능 | 설명 |
|---|------|------|
| 1 | 요소 선택 | 클릭 → 하이라이트 + 피드백 입력 |
| 2 | 텍스트 선택 | 드래그 → 선택 문자열 캡처 |
| 3 | 영역 선택 | 빈 공간 드래그 → 바운딩 박스 캡처 |
| 4 | 다중 선택 | Shift+클릭으로 여러 요소 묶기 |
| 5 | 애니메이션 일시정지 | CSS 애니메이션 동결/재개 |
| 6 | 마크다운 출력 | 4단계 상세도 (Compact/Standard/Detailed/Forensic) |
| 7 | 키보드 단축키 | 오브젝트 기반 수정자 키 지원 |
| 8 | 단축키 설정 팝업 | 중앙 오버레이 + 드래프트 패턴 |
| 9 | 피드백 팝업 드래그 | 헤더 드래그로 팝업 이동 |
| 10 | 검수 OFF 시 출력 | 피드백 있으면 모드 무관 동작 |
| 11 | sessionStorage 저장 | 새로고침해도 피드백 유지 |
| 12 | 피드백 유형 분류 | 🎨 UI / ⚙️ 기능 / 📝 텍스트 |
| 13 | 피드백 편집/삭제 | 뱃지 클릭 → 수정 팝업 |

#### 기본 단축키

## 2. 문제점 / 개선 필요성

### 2-1. 파이프라인 단절
현재 파이프라인에서 사람이 직접 해야 하는 수작업 단계가 많음:

```
[수작업] 어떤 페이지를 검수할지 결정
[수작업] 브라우저에서 각 페이지를 하나씩 열기
[도구]   QA 도구로 피드백 입력
[수작업] 마크다운 복사
[수작업] DEV-REQUEST 형식에 맞춰 정리
[수작업] 클로드 코드에 전달
[수작업] 수정 완료 후 다시 확인
```

### 2-2. PRD 기반 자동 검증 부재
PRD에 "이런 기능이 있어야 한다"고 정의해도, 실제 구현 여부는 사람이 하나씩 확인해야 함.
코워커(Cowork)가 PRD 대비 코드를 자동으로 비교하는 체계가 없음.

### 2-3. 검수 결과의 추적성 부족
- 피드백이 sessionStorage에만 저장 → 탭 닫으면 사라짐
- 이전 검수 이력 조회 불가
- "이 버그는 언제 발견했고 언제 고쳐졌는지" 추적 불가

### 2-4. 데모/증빙 기능 부재
- 검수 과정이나 결과를 시각적으로 보여줄 방법 없음
- 클라이언트/팀원에게 진행 상황 공유가 어려움

## 3. 목표

> **핵심 목표**: PRD → 개발 → 검수 → 수정 → 재검수의 전체 사이클을 더 효율적으로 만들기

### 3-1. 검수 파이프라인 자동화
- 검수 대상 페이지 자동 식별 (PRD에서 추출)
- 검수 결과 → DEV-REQUEST 자동 생성
- 수정 후 재검수 워크플로우

### 3-2. PRD 기반 체크리스트 자동 생성
- PRD의 기능 요구사항을 파싱하여 검수 체크리스트 자동 생성
- 체크리스트 항목별 통과/실패 기록

### 3-3. 검수 이력 관리
- 검수 세션별 기록 저장 (로컬 파일 또는 DB)
- 이전 검수 결과 조회 및 비교
- 버그 발견→수정→재확인 라이프사이클 추적

### 3-4. 검수 결과 시각화/공유
- 검수 리포트 자동 생성 (마크다운/HTML)
- 데모 영상 또는 스크린샷 기반 증빙

## 5. 기술적 고려 사항

### 5-1. 현재 아키텍처 유지
- 순수 JS 단일 파일 (외부 의존성 없음)
- IIFE 패턴, 전역 오염 없음
- 기존 프로젝트 코드에 영향 없는 독립 도구

### 5-2. 범용성
- marketing-automation뿐 아니라 모든 프로젝트에서 사용 가능
- `claude-global` 레포에 포함하여 프로젝트 셋업 시 자동 배포
- React 프로젝트: Agentation과의 역할 분리/통합 검토

### 5-3. 코워커(Cowork) 연동
- 코워커가 검수 결과를 읽고 DEV-REQUEST를 작성하는 워크플로우
- 코워커 ↔ QA 도구 간 데이터 연동 방법 논의 필요
